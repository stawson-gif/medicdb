# Развёртывание МедСистемы с SQLite

Приложение не требует Supabase или отдельного сервера базы данных. SQLite и загруженные файлы хранятся на одном постоянном диске.

## Railway

1. Создайте сервис из GitHub-репозитория.
2. Откройте сервис → **Variables** и добавьте:
   - `DATABASE_URL=/data/med.db`
   - `FILES_DIR=/data/uploads`
   - `JWT_SECRET=<длинная случайная строка>`
3. Откройте сервис → **Volumes** → **Add Volume**.
4. Укажите Mount Path: `/data`.
5. В **Settings → Networking** создайте публичный домен.
6. Выполните Redeploy.

Таблицы создаются автоматически при первом обращении. Отдельная команда миграции не нужна.

## Локальный запуск

Скопируйте `.env.example` в `.env`, затем выполните:

```bash
npm install
npm run dev
```

База будет находиться в `data/med.db`, вложения — в `data/uploads`.

## Важно

- Не разворачивайте эту SQLite-версию на Vercel: файловая система serverless-функций не предназначена для постоянного хранения.
- Делайте резервную копию всей директории `/data`, а не только файла `med.db`.
- Редактирование поддерживается для `.docx`. Старый бинарный формат `.doc` можно сохранить и скачать, но нельзя открыть в редакторе без LibreOffice/OnlyOffice.
- При преобразовании сложная Word-вёрстка, колонтитулы, макросы и некоторые изображения могут переноситься не полностью. Оригинал всегда сохраняется отдельно.
